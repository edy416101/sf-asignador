⚙ Configuración de campos

📋 Campos base del pre-template. Por defecto usan valores hardcodeados (el dropdown de la UI, "DOCTOR", "Mexico", etc). Puedes cambiar cualquiera para que traiga el valor real del Contact desde Salesforce. Útil cuando te especifican "usa el ContactTypeSegment y FacilitySize que ya tiene el Contact". La validación verifica que el campo exista en el objeto Contact antes de aceptar.
Campos adicionales
➕ Campos custom adicionales. Agregan columnas nuevas al pre-template. Elige la fuente del valor:
· ☁ Desde SF — trae el valor del Contact real (útil para Phone, MobilePhone, etc).
· ✎ Manual — tú escribes el valor, se estampa idéntico en todos los registros (útil para Suffix, campaign codes, códigos de tracking, etc).

📥 Cargar datos manualmente

Los headers deben ser los API Field Names exactos de Salesforce. El orden es libre. Los headers reconocidos se mapean a las columnas del pre-template. Los headers que no matchean con las columnas base (13 campos) pero tienen sintaxis válida se agregan automáticamente como custom fields y aparecerán en el Field Mapping. Columnas con nombre inválido (espacios, puntos) o que empiezan con __ se ignoran.
